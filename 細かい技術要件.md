# 歴史人物AI相談所：詳細開発・技術仕様

本プロジェクトにおける、FlutterとFirebase、およびGemini AIを活用した「没入型対話アプリ」の開発・運用スタックを定義します。

## 1. フロントエンド構成（Flutter）
*   **状態管理（State Management）**: `Riverpod` (v2 アノテーション推奨) を採用。1.5画面という特殊なUIにおける状態共有とリアクティブなUI更新を効率化するため。
*   **UI/UXアニメーション**:
    *   `flutter_hooks` を併用し、タイピングエフェクトや感情オーラ（ShaderMask/CustomPainter）の複雑なアニメーション制御を簡潔に記述。
    *   `vibration` パッケージによる歴史的衝撃（現代語への戸惑い等）のフィードバック実装。
*   **ストレージ**: Firestoreとのリアルタイム同期を基本とし、`isar` 等の高性能ローカルDBでのキャッシュ保管により、オフライン閲覧も部分的にサポート。

## 2. バックエンド・インフラストラクチャ（Firebase）
*   **認証（Firebase Auth）**:
    *   初回は「匿名ログイン」により、オンボーディングなしで即座に相談体験を開始。
    *   ユーザー属性（好みや親密度ランク）をFirestoreに自動生成。
*   **データベース設計（Cloud Firestore）**:
    *   **`/characters/{charId}`**: 各偉人の性格パラメータ（4層プロンプト）、ビジュアルパス、音声トーン設定（将来拡張用）。
    *   **`/users/{uid}/conversations/{convId}`**: 対話スレッド管理。Geminiへのコンテキスト供給用。
    *   **`/users/{uid}/archive/modern_words`**: 偉人が会話で見つけた「現代語辞典」のコレクション進捗。
*   **ロジック処理（Cloud Functions for Firebase）**:
    *   Gemini APIとのセキュアな通信基盤。
    *   好感度スコアに応じた称号（ランク）の自動更新やイベント発生トリガーの計算。

## 3. AIエンジン連携（Gemini 3 シリーズ）
*   **Vertex AI for Firebase**:
    *   Firebase SDK経由でGemini Pro（詳細相談）とFlash（リアクション・雑談）を論理的に使い分ける。
*   **プロンプト・エンジニアリング**:
    *   **「4層構造プロンプト」**: 「システム（コア行動）」「感情（状態変化）」「ナレッジ（歴史的背景）」「現代適正（戸惑いの強度）」を動的に合成して送信。
    *   **Function Calling**: 歴史人物が「現代語」を認識した際に、それをアプリの「辞典システム」に登録するイベントを抽出。

## 4. エージェント駆動開発（Antigravity）への指示
*   **コンポーネント指向**: エージェントに対し、1.5画面構成を分割した「再利用可能なWidget単位」での出力と、その自動Widgetテストの実行を指示する。
*   **モック環境**: バックエンド未接続時でもUIアニメーションとモックデータによる検証を可能にするため、DI（Dependency Injection）パターンを徹底する。
